# P2P 文件共享

一个完全去中心化的文件共享网站，使用 WebRTC 实现点对点传输。

## ✨ 特性

- 🔒 **完全私密** - 文件直接在设备间传输，不经过任何服务器
- 🚀 **零配置** - 无需注册、登录或安装
- 📱 **跨平台** - 支持所有现代浏览器（手机/电脑）
- 🔐 **端到端加密** - WebRTC 内置 DTLS 加密
- 📤 **多文件支持** - 可同时传输多个文件
- 💻 **响应式设计** - 完美适配手机和电脑

## 🚀 使用方法

### 快速开始

1. **在电脑上打开网站**
   - 访问网站地址
   - 选择"发起连接"标签
   - 屏幕上会显示二维码和6位连接码

2. **在手机上连接**
   - 同样打开网站
   - 选择"加入连接"标签
   - 扫描二维码 或 手动输入连接码
   - 点击"加入连接"

3. **开始传输**
   - 连接成功后，可以拖拽文件或点击选择文件
   - 文件会直接传输到对方设备
   - 接收的文件会自动下载

### 使用场景

- 📱 手机 → 电脑：传输照片、文档等
- 💻 电脑 → 手机：发送文件到手机
- 🖥️ 电脑 → 电脑：同网络下快速传文件

## 🛠 技术实现

### 核心技术

- **WebRTC** - 实时点对点通信
- **RTCPeerConnection** - 建立 P2P 连接
- **RTCDataChannel** - 传输文件数据
- **BroadcastChannel API** - 信令交换（同一浏览器内）
- **QRCode** - 二维码生成
- **jsQR** - 二维码扫描识别

### 文件结构

```
├── index.html      # 主页面
├── style.css       # 样式表
├── app.js          # 应用逻辑
└── README.md       # 说明文档
```

### 架构说明

```
┌─────────────┐                    ┌─────────────┐
│   电脑浏览器   │ ◁── WebRTC ──▷ │   手机浏览器   │
│             │   P2P 通道        │             │
│  (发起方)    │ ◁── 文件流 ──▷  │  (加入方)    │
└─────────────┘                    └─────────────┘
```

**信令机制**：
- 当前实现使用 BroadcastChannel API 进行信令交换
- 适用于同一浏览器标签页间的通信
- 如需跨设备（不同 IP）使用，需要部署信令服务器

## 📦 部署

### 本地测试

```bash
# 使用 Python 启动本地服务器
python3 -m http.server 8080

# 或使用 Node.js
npx serve .
```

然后访问 `http://localhost:8080`

### 静态托管

可以部署到任何静态网站托管服务：
- GitHub Pages
- Vercel
- Netlify
- Cloudflare Pages

### 跨设备使用说明

当前版本使用 BroadcastChannel API，**仅适合同一浏览器内的标签页通信**。

要实现真正的跨设备（手机到电脑）传输，需要：

1. **部署信令服务器**（WebSocket）
2. **修改 `app.js` 中的 `setupSignaling()` 函数**
3. **或使用 WebRTC 的其他方式交换 SDP**

#### 替代方案（无需服务器）

用户可以手动交换 SDP 信息：
1. 复制电脑端的 SDP Offer
2. 粘贴到手机端
3. 手机生成 SDP Answer
4. 粘贴回电脑端
